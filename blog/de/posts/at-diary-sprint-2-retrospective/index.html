Sprint 2 von @AT_Diary ist abgeschlossen — Build erfolgreich. Hier ist, was wir geliefert haben.

Was ist @AT_Diary?

@AT_Diary ist eine datenschutzorientierte iOS-Tagebuch-App, die täglich deinen Lebenskontext automatisch in ein strukturiertes Tagebuch verwandelt. Kein Server. Kein Cloud-Speicher. Keine Daten verlassen dein Gerät.

Die App zieht Kontext aus vier nativen Apple-Frameworks und verknüpft diese zu einer kohärenten Tageserzählung — vollständig auf dem Gerät.

Wichtige Technische Entscheidungen

Warum ausschließlich native Frameworks?
Sobald Daten ein Netzwerk berühren, braucht man eine Datenschutzrichtlinie, einen zu sichernden Server und eine Vertrauensbeziehung mit dem Nutzer, die leicht zerstört werden kann. Nur nativ bedeutet: Das Tagebuch ist genauso privat wie das iPhone selbst.

Keine Drittanbieter-Abhängigkeiten.
Jede Bibliothek stammt von Apple. Das bedeutet, unsere Update-Oberfläche ist identisch mit der iOS-Update-Oberfläche.

On-Device-Syntheseschleife:

Kontext abrufen (MusicKit / EventKit / CoreLocation / Photos)
    → Aggregieren nach Zeitfenster
    → Strukturierten Tagebucheintrag generieren
    → Auf verschlüsseltem On-Device-Speicher speichern

Was als Nächstes kommt

Sprint 3 konzentriert sich auf:

  • Zusammenfassung in natürlicher Sprache — On-Device-Modelle wandeln Rohdaten in lesbare Tagebuchprosa um
  • Tages-Digest-UI — eine übersichtliche, zeitlinienbasierte Ansicht für vergangene Einträge
  • Export-Schicht — verschlüsselter PDF/Markdown-Export vollständig auf dem Gerät

Architekturprinzip

„Das Tagebuch soll privater sein als ein Papiernotizbuch. Papier kann gefunden werden. Dieses nicht."

Die gesamte Anwendung ist darauf ausgelegt, dass keine Daten jemals das Gerät des Nutzers verlassen.
