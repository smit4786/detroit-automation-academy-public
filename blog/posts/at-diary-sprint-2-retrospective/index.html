What Is @AT_Diary?

@AT_Diary is a privacy-first iOS journaling application that automatically synthesizes your daily life into a structured diary. No servers. No cloud storage. No data leaves your device.

The application pulls context from four native Apple frameworks and weaves them into a coherent daily narrative — entirely on-device.

Key Technical Decisions

Why native frameworks only?
The moment data touches a network, you need a privacy policy, a server to secure, and a trust relationship with the user that’s easy to destroy. Native-only means the diary is as private as the iPhone itself.

No third-party dependencies.
Every library is Apple’s. This means our update surface is identical to iOS’s update surface — when Apple ships, we stay current automatically.

On-device synthesis loop:

fetch context (MusicKit / EventKit / CoreLocation / Photos)
    → aggregate by time window
    → generate structured diary entry
    → persist to on-device encrypted store

What’s Next

Sprint 3 will focus on:

  • Natural Language Summarization — converting raw event aggregates into readable diary prose using on-device Models
  • Daily Digest UI — a clean, timeline-based view for browsing past entries
  • Export Layer — encrypted PDF/Markdown export fully on-device

Architecture Principle

“The diary should be more private than a paper journal. Paper can be found. This cannot.”

The entire application is designed around the assumption that zero data ever leaves the user’s device — not during diary creation, not during storage, not during export.

Sprint 2 delivered the framework integration layer that makes this promise real.
