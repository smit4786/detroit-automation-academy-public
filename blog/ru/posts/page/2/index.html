Запуск регистрации в Detroit Automation Academy: от нуля до Cloud Run за одну сессию

Сегодня мы перевели систему регистрации Detroit Automation Academy из состояния сломанной сборки в полностью развернутую и доступную систему всего за одну инженерную сессию. Вот как это происходило.

Начальная точка: пять сбоев сборки подряд

Скрипт deploy.sh был. Образы Docker были. Чего не было, так это работающей сборки.

Первой ошибкой была синтаксическая ошибка Go в backend/main.go — символ новой строки внутри строкового литерала:

// Ошибка
fmt.Printf("Detroit Automation Academy Registration Backend starting on port %s...
", port)

// Исправлено
fmt.Printf("Detroit Automation Academy Registration Backend starting on port %s...
", port)

Go не допускает неэкранированных переносов строк внутри строковых литералов. Исправление заняло один символ.

Основная концепция

Большинство приложений для ведения дневников просят вас доверить их серверам ваши самые личные данные. AT_Diary этого не делает. Каждая единица данных, с которой он работает — ваша история музыки, события календаря, контекст местоположения, ваши фотографии — обрабатывается на устройстве и хранится с шифрованием iCloud. Никакие данные не отправляются вовне.

Исправление нашего CI/CD конвейера: подмодули, пустые коммиты и QA-гейт Clawd

Блог Automated Insights столкнулся с повторяющейся проблемой CI/CD: развертывания, которые работали локально, незаметно давали сбой в GitHub Actions или завершались успешно, но приводили к поломке сайта. Вот анализ основных причин и внесенные нами изменения.

Три режима отказа

  1. Сборка Hugo успешна, контент сайта отсутствует — директория public/ была создана, но шаг git commit завершился неудачей, так как нечего было коммитить. Шаг развертывания все равно запустился и отправил устаревшую сборку.

  2. Подмодуль темы не инициализирован — тема Ananke является подмодулем git. Новые раннеры GitHub Actions клонируют репозиторий, но не инициализируют подмодули автоматически. Сборка продолжалась с пустой директорией themes/ananke/, в результате чего получался сайт без стилей, хотя код завершения был 0.
