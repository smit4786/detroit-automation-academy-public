Блог Automated Insights столкнулся с повторяющейся проблемой CI/CD: развертывания, которые работали локально, незаметно давали сбой в GitHub Actions или завершались успешно, но приводили к поломке сайта. Вот анализ основных причин и внесенные нами изменения.

## Три режима отказа

1. **Сборка Hugo успешна, контент сайта отсутствует** — директория `public/` была создана, но шаг git commit завершился неудачей, так как нечего было коммитить. Шаг развертывания все равно запустился и отправил устаревшую сборку.

2. **Подмодуль темы не инициализирован** — тема Ananke является подмодулем git. Новые раннеры GitHub Actions клонируют репозиторий, но не инициализируют подмодули автоматически. Сборка продолжалась с пустой директорией `themes/ananke/`, в результате чего получался сайт без стилей, хотя код завершения был 0.

3. **Задержка инвалидации кэша** — страницы, кэшированные в CDN, оставались устаревшими после нового развертывания.

## Исправление 1: Инициализация подмодулей

```yaml
- uses: actions/checkout@v4
  with:
    submodules: true        # ← этого не хватало
    fetch-depth: 0
```

Без `submodules: true` Hugo находит `theme = "ananke"`, ищет в `themes/ananke/layouts/`, ничего не находит и откатывается к шаблонам по умолчанию — полностью без стилей.

## Исправление 2: Защита от пустых коммитов

```yaml
# До — всегда коммитит, выдает ошибку, если ничего не изменилось
- run: |
    git add public/
    git commit -m "Deploy"
    git push

# После — пропускает, если изменений нет
- run: |
    git add public/
    git diff --staged --quiet || git commit -m "chore: deploy $(date -u +%Y-%m-%dT%H:%M:%SZ)"
    git push
```

`git diff --staged --quiet` завершается с кодом 0 (нет изменений) → пропустить коммит. Завершается с ненулевым кодом (есть изменения) → коммит и пуш.

## QA-гейт Clawd

Поскольку предприятие Automated Technologies работает на базе фреймворка автономных агентов Clawd, мы настроили агента `qa_gate.py` для автоматической проверки развертываний:

1. Запрашивает URL живого сайта после развертывания.
2. Проверяет наличие заголовка последнего поста в HTML.
3. Проверяет, что заголовок ответа `Last-Modified` новее, чем временная метка предыдущего развертывания.
4. Прерывает конвейер, если любая из проверок не пройдена.

Это превращает проверку развертывания из ручного процесса в автоматизированный гейт — в соответствии с принципами работы всех агентов Clawd.

## Текущее состояние

Теперь конвейер работает надежно. Оставшиеся задачи:

- **Многоязычные посты** — переводов на немецкий и русский языки мало. Английский контент является авторитетным.
- **Поиск** — оценка Pagefind (клиентский поиск без зависимостей) для будущих спринтов.
