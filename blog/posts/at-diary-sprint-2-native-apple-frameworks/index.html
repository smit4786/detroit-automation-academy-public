@AT_Diary Sprint 2: Building a Privacy-First Diary on Apple Native Frameworks

Justin Smith

February 28, 2026 - 2 minutes read - 403 words

Sprint 2 of @AT_Diary is complete with a successful build. Here’s what we shipped and how the architecture decisions are holding up.

The Core Premise

Most journaling apps ask you to trust their servers with your most personal data. AT_Diary doesn’t. Every piece of data it touches — your music history, your calendar events, your location context, your photos — is processed on-device and stored with iCloud encryption. Nothing phones home.

This constraint drives every architectural decision.

The Framework Stack

We use five Apple-native frameworks, each responsible for a distinct data domain:

FrameworkData DomainPrivacy Approach
MusicKitWhat you’re listening toRequests user permission; no data leaves device
EventKitCalendar & remindersRead-only access to authorized calendars
ContactsNamed entities in your dayAccessed by identifier only, never bulk imported
CoreLocationWhere you areSignificant location changes only, stored as geohash
PhotosVisual memory triggersThumbnails generated locally, originals never read

The synthesis layer combines these streams into a structured diary entry — automatically, without the user needing to write a word.

Sprint 2 Deliverables

✅ MusicKit Integration

MusicKit authorization flow is complete. The app requests permission on first launch and uses the MusicPlayer.shared.queue API to capture the current track at entry creation time. Album artwork is pulled from the MusicKit catalog and displayed as a contextual thumbnail.

✅ EventKit Read Pipeline

Calendar events for the current day are fetched from EKEventStore and ranked by duration and proximity to the current time. The top 3 events populate the “Today’s Context” section of a diary entry automatically.

✅ CoreData Persistence

All diary entries are stored in a CoreData stack with CloudKit sync enabled via NSPersistentCloudKitContainer. This means your entries are available across all your Apple devices without ever touching a third-party server.

✅ Successful Build

The project builds clean against iOS 18.0 target with zero warnings on the privacy-sensitive framework calls.

What’s Next: Sprint 3

Sprint 3 focuses on the user interface — moving from a functional data pipeline to an experience that feels as polished as the underlying architecture:

  • Entry card design: Rich contextual tiles showing music, location, and event context
  • Timeline view: Chronological scroll through entries with ambient background colors derived from album art
  • On-device ML: Using CreateML to surface recurring patterns in your diary (most common locations, frequent collaborators, emotional tone from writing)

@AT_Diary is a private project under Automated Technologies. No App Store release is planned at this time.
