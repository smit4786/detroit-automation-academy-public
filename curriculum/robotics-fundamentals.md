# Robotics Fundamentals

**Learning Track:** Detroit Automation Academy  
**Language:** English  
**Level:** Beginner (Grades 6-8)  
**Duration:** 8 weeks (16 contact hours)  
**Last Updated:** April 19, 2026

---

## Course Overview

Robotics Fundamentals introduces students to the core principles of robotics, automation, and mechanical engineering. Through hands-on projects and real-world applications, students learn to design, build, and program basic robotic systems while developing critical problem-solving skills.

This course serves as a foundation for advanced automation and engineering pathways.

---

## Learning Objectives

By completing this module, students will be able to:

### Knowledge
- [ ] Understand the basic components and systems of a robotic platform
- [ ] Identify different types of robots and their real-world applications
- [ ] Explain the role of sensors, actuators, and controllers in robotics
- [ ] Describe basic principles of mechanics and physics in robotics

### Skills
- [ ] Assemble and troubleshoot basic robotic platforms
- [ ] Write and debug simple robot control programs
- [ ] Design and implement mechanical solutions to engineering problems
- [ ] Conduct systematic testing and iteration on robotic designs

### Attitudes
- [ ] Develop persistence and resilience when troubleshooting
- [ ] Collaborate effectively in team-based engineering projects
- [ ] Think computationally about real-world problems
- [ ] Appreciate the role of robots in modern industry

## Module Structure

### Lesson 1: Introduction to Robotics
**Duration:** 2 hours  
**Objective:** Understand what robots are and their role in society

**Topics:**
- Definition of robotics and automation
- History of robotics
- Types of robots (industrial, service, autonomous)
- Real-world applications in manufacturing, healthcare, and exploration
- Ethical considerations in automation

**Activities:**
- Explore videos of robots in action
- Discuss: Where do robots exist in Detroit?
- Brainstorm: What problems could robots solve?

**Key Concepts:**
- Automation increases efficiency and precision
- Robots are designed for specific tasks
- Robotics combines mechanics, electronics, and programming

---

### Lesson 2: Basic Robotics Components
**Duration:** 2 hours  
**Objective:** Learn the building blocks of robotic systems

**Topics:**
- **Mechanical Components:**
  - Motors and actuators
  - Gears, pulleys, and levers
  - Wheels, axles, and frames
  - Joint types and degrees of freedom

- **Electrical Components:**
  - Power sources (batteries)
  - Switches and relays
  - Wiring and connections
  - Safety considerations

- **Computational Components:**
  - Microcontrollers
  - Sensors and inputs
  - Programming logic
  - Control algorithms

**Activities:**
- Hands-on: Identify components in sample robots
- Build: Assemble a simple gear system
- Experiment: Test different motor configurations

**Key Concepts:**
- Robots consist of three integrated systems: mechanical, electrical, computational
- Each component has specific roles and limitations
- System design requires understanding component interactions

---

### Lesson 3: Sensors and Data Collection
**Duration:** 2 hours  
**Objective:** Understand how robots perceive their environment

**Topics:**
- Types of sensors (touch, light, sound, distance, temperature)
- How sensors convert physical phenomena to electronic signals
- Sensor calibration and accuracy
- Data interpretation and thresholds
- Feedback loops in robotic systems

**Activities:**
- Experiment: Test different sensors in various conditions
- Calibrate: Set up and adjust sensor thresholds
- Debug: Troubleshoot sensor readings
- Program: Write sensor-based conditional logic

**Key Concepts:**
- Sensors enable robots to perceive and respond to their environment
- Sensor data quality affects robot performance
- Feedback is essential for adaptive robot behavior

---

### Lesson 4: Programming Robot Behavior
**Duration:** 2 hours  
**Objective:** Learn to write code that controls robots

**Topics:**
- Robot programming frameworks (block-based and text-based)
- Control structures (sequences, conditionals, loops)
- Motor control and speed regulation
- Sensor-based decision making
- Debugging robot behavior
- Documentation and commenting

**Activities:**
- Code: Write programs to move robot in patterns
- Challenge: Program robot to navigate a course
- Debug: Identify and fix logic errors
- Optimize: Improve program efficiency

**Key Concepts:**
- Robot behavior is determined by programmed instructions
- Sensor data enables adaptive programming
- Testing and iteration are essential for robot success

---

### Lesson 5: Mechanical Design and Build
**Duration:** 3 hours  
**Objective:** Design and construct functional robotic platforms

**Topics:**
- Basic engineering design process
- Structural stability and balance
- Power transmission and efficiency
- Tool usage and safety
- Assembly techniques and best practices
- Troubleshooting mechanical problems

**Activities:**
- Design: Sketch robotic platform for specific task
- Build: Assemble frame and drivetrain
- Test: Verify structure can support intended load
- Iterate: Modify design based on testing results
- Document: Record design decisions and improvements

**Key Concepts:**
- Good mechanical design enables reliable robot function
- Stability, strength, and efficiency are interconnected
- Iterative design improves performance

---

### Lesson 6: Autonomy and Decision Making
**Duration:** 2 hours  
**Objective:** Program robots to make independent decisions

**Topics:**
- Autonomous vs. remote-controlled behavior
- State machines and behavior logic
- Multi-sensor decision making
- Error handling and recovery
- Real-world constraints (battery life, communication delays)
- Testing autonomous systems

**Activities:**
- Program: Create state machine for robot behavior
- Test: Verify autonomous operation in various scenarios
- Challenge: Program robot to complete complex autonomous task
- Analyze: Evaluate success and failure modes

**Key Concepts:**
- Autonomous robots require robust decision logic
- Multi-sensor systems provide redundancy and reliability
- Testing is critical before deploying autonomous systems

---

### Lesson 7: Teamwork and Project Integration
**Duration:** 2 hours  
**Objective:** Apply skills to team-based robotics projects

**Topics:**
- Team roles and responsibilities
- Collaborative design and problem-solving
- Project planning and time management
- Integration of mechanical, electrical, and computational systems
- Communication within technical teams
- Presentation of technical work

**Activities:**
- Project: Team designs and builds robot for specific challenge
- Present: Demonstrate robot capabilities and design decisions
- Evaluate: Peer review of other team projects
- Reflect: Discuss what worked and what could improve

**Key Concepts:**
- Real-world robotics requires multidisciplinary teamwork
- Effective communication is essential for complex projects
- Iterative improvement through feedback

---

### Lesson 8: Advanced Topics and Future Pathways
**Duration:** 1 hour  
**Objective:** Explore advanced robotics concepts and career pathways

**Topics:**
- Computer vision and image processing
- Machine learning in robotics
- Swarm robotics and multi-robot systems
- Robotics in industry 4.0
- Career pathways in robotics and automation
- Further learning resources

**Activities:**
- Explore: Research advanced robotics applications
- Discuss: Career opportunities in automation
- Reflect: Personal interests and next steps
- Plan: Identify advanced courses or projects

**Key Concepts:**
- Robotics is a rapidly evolving field with emerging technologies
- Strong fundamentals enable learning advanced topics
- Robotics skills are in high demand

## Hands-On Projects

### Project 1: Build and Program a Basic Mobile Robot
**Duration:** 3 hours  
**Objective:** Apply mechanical, electrical, and programming skills

**Requirements:**
- Build frame and drivetrain
- Install motors, wheels, and sensors
- Program forward/backward/turn movement
- Test navigation capability

**Success Criteria:**
- Robot moves smoothly in straight lines
- Robot turns when commanded
- Sensor reading displays on display
- Code is well-documented

---

### Project 2: Autonomous Navigation Challenge
**Duration:** 2 hours  
**Objective:** Program autonomous navigation through obstacles

**Requirements:**
- Use ultrasonic sensor to detect obstacles
- Program autonomous obstacle avoidance
- Navigate from start to goal without remote control
- Complete within time limit

**Success Criteria:**
- Robot navigates autonomously without human control
- Successfully avoids obstacles
- Reaches goal location
- Completes in under time limit

---

### Project 3: Gripper/Manipulator Design
**Duration:** 2 hours  
**Objective:** Design and build a robotic gripper

**Requirements:**
- Design gripper mechanism
- Implement mechanical advantage with gears/pulleys
- Program grip and release commands
- Pick up objects of varying size/weight

**Success Criteria:**
- Gripper securely holds objects
- Controlled grip strength
- Reliable pickup/release
- Withstands repeated use

## Assessment Methods

### Formative Assessment (Throughout Course)
- Observation of hands-on activities
- Questioning during lessons
- Peer feedback on project work
- Self-reflection and journaling

### Summative Assessment (End of Course)
- **Practical Project (50%):** Successful completion of capstone robot challenge
- **Knowledge Test (25%):** Multiple-choice and short-answer on concepts
- **Technical Documentation (15%):** Well-commented code and design documentation
- **Teamwork & Communication (10%):** Collaboration and presentation skills

### Grading Scale
- 90-100%: Advanced (A)
- 80-89%: Proficient (B)
- 70-79%: Developing (C)
- Below 70%: Needs Support (I)

## Safety Considerations

### Physical Safety
- [ ] Students wear work gloves when building
- [ ] Teachers supervise tool usage (screwdrivers, etc.)
- [ ] Electrical cords kept organized and untangled
- [ ] No running with robots or equipment
- [ ] Proper handling of batteries and power sources

### Electrical Safety
- [ ] All electrical connections properly insulated
- [ ] Batteries checked for damage before use
- [ ] Short circuit hazards identified and mitigated
- [ ] Emergency power-off procedures established

### Software Safety
- [ ] Anti-malware on computers
- [ ] Code reviewed before deployment
- [ ] Emergency stop function programmed
- [ ] Backup battery limits enforced in software
